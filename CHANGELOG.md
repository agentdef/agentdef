- Fixed: `extract_bullets()` in `importers/_common.py` only recognized `-`/`*` unordered lists, so a source `## Priorities` section written as an ordered list (`1. ...`, `2. ...`) fell through to the raw-line fallback with its original numbering intact — `AgentfileWriter._render_agent_md()` then added its *own* numbering on top, producing doubled markers like "1. 1. Security issues...". Now recognizes and strips ordered-list markers (`1.` / `1)`) too. Found while building the Claude → Copilot demo below; regression tests added.

- New: [`docs/dashboard-tour.md`](docs/dashboard-tour.md) — guide to the `understand-dashboard/` knowledge-graph explorer generated for this repo, with a two-minute guided path through the adapter/importer architecture.
- Reverse adapters ("importers"): convert an existing framework agent definition into an Agentfile directory
  - `importers/claude` — imports `CLAUDE.md` (high-fidelity round-trip for Agentfile-generated files) and Claude Code subagent files (`.claude/agents/<name>.md`-style, with YAML frontmatter: `name`/`description`/`tools`)
  - `importers/copilot` — imports `.github/copilot-instructions.md`, path-scoped `.github/instructions/*.instructions.md`, and custom `NAME.agent.md` agents
  - `importers/m365copilot` — imports Microsoft 365 Copilot declarative agent manifests (schema v1.0–v1.7)
  - `importers/copilotstudio` — imports Microsoft Copilot Studio agent docs (as published in github/awesome-copilot-studio-agents); extracts the fenced "Instructions" code block (the real system prompt) and classifies it via the shared markdown importer, mapping Conversation Starters, Knowledge Sources, and Deployment Notes to their Agentfile equivalents
  - Shared deterministic markdown classifier, YAML frontmatter parser, and `AgentfileWriter` in `importers/_common.py`
  - Every import run produces an `IMPORT_REPORT.md` (mapped / inferred / dropped)
  - Validated against real-world corpora, 0 failures across all four importers after fixing five real bugs found along the way:
    - 219 agents from [github/awesome-copilot](https://github.com/github/awesome-copilot/tree/main/agents) (`copilot` importer)
    - 94 agents from [github/awesome-copilot-studio-agents](https://github.com/github/awesome-copilot-studio-agents) (`copilotstudio` importer)
    - 187 agents from [github/awesome-claude-agents](https://github.com/vijaythecoder/awesome-claude-agents) and [github/awesome-claude-code-subagents](https://github.com/VoltAgent/awesome-claude-code-subagents) (`claude` importer, subagent shape)
    - Bugs fixed:
      - Markdown headers were parsed inside fenced code blocks, so a `#`-prefixed shell/Python/YAML comment in an example snippet (very common in DevOps-flavored agent docs) could shred the section structure. `split_sections()` now tracks fence state (`` ``` `` / `~~~`) and ignores headers while inside one.
      - The Role-inference "You are ..." sentence detector didn't cross a soft line-wrap, so hard-wrapped prose silently produced no match; it now spans wrapped lines (up to a real paragraph break) and also recognizes "Your role is ...", "You act as ...", "You serve as ...", and "You help ..." phrasings.
      - Docs with no leading preamble and no persona sentence anywhere used to fall back to a bare "(role not specified in source)" placeholder even when a perfectly good description sentence was sitting in the first section; Role now falls back to that section's first sentence instead.
      - **YAML frontmatter silently dropped on parse error**: a plain-scalar frontmatter value containing its own unescaped `": "` later in the line (e.g. `description: ... Triggers on: 'GDPR', 'CCPA', ...`) makes `yaml.safe_load` raise `ScannerError`, which each importer's local frontmatter parser caught with a broad `except Exception: return {}, text`. That silently discarded the entire frontmatter block — it then leaked verbatim into `instructions/core.md` as ordinary preamble text instead of populating `name`/`description`/`tools`. Affected 17/94 `copilotstudio` files and 9/187 `claude` subagent files (0/219 `copilot` files, by luck of the corpus). Fixed by consolidating the three near-duplicate frontmatter parsers into one shared `split_frontmatter()` in `_common.py`, which now falls back to a lenient line-based parser instead of giving up when strict YAML parsing fails.
      - **Non-standard frontmatter closing fence**: some source files close the frontmatter block with a long run of dashes (a markdown thematic break) instead of exactly `---`, which the old regex didn't recognize, leaving the whole block unparsed. The frontmatter delimiter regex now accepts any run of 3+ dashes for both the opening and closing fence, per markdown's own thematic-break convention.
- `agentfile` CLI: a single console command (`pip install -e .` from `agentfile/`) wrapping validate/adapt/import — `agentfile validate`, `agentfile adapt <framework>`, `agentfile import <framework>`, `agentfile list`. Currently works from an editable git-checkout install only; a self-contained PyPI package is planned (see PUBLISHING.md).
